# Чек-лист выполнения итогового проекта

## 1. Firebase Authentication (4 балла)
- [ ] **Регистрация и вход (2 балла):** реализованы экраны регистрации и входа с использованием FirebaseAuth.
- [ ] **Взаимодействие с почтой (2 балла):** реализован сброс пароля через email **или** подтверждение регистрации через email (верификация).

## 2. Firebase Firestore / Realtime Database (3 балла)
- [ ] **Хранение данных (3 балла):** данные приложения (пользовательские записи, настройки и т.д.) сохраняются в Firestore или Realtime Database.

## 3. Соблюдение темы (5 баллов)
- [x] **Полнота реализации темы (5 баллов):** приложение полностью выполняет основную задачу выбранной темы (например, для трекера финансов – учёт доходов/расходов, для планировщика путешествий – создание поездок и т.д.).

## 4. Уведомления (4 балла)
- [ ] **Локальные/пуши (4 балла):** реализованы уведомления (например, напоминания, предупреждения), которые приходят в нужный момент и соответствуют логике приложения.

## 5. Архитектура (4 балла)
- [x] **MVVM или Clean Architecture (4 балла):** код организован с использованием одного из этих подходов (чёткое разделение на слои, отделение логики от UI).

## 6. Анимации (2 балла)
- [x] **Логичные анимации (2 балла):** добавлены анимации загрузки, переходов между экранами, появления элементов – единообразно во всём приложении.

## 7. Управление состоянием через Bloc/Cubit (4 балла)
- [x] **Bloc/Cubit (4 балла):** всё состояние приложения управляется через Bloc или Cubit (нет setState, InheritedWidget и т.п. для глобального состояния).

## 8. Оформление и интерфейс (4 балла)

### 8.1 Общий дизайн и стиль (1 балл)
- [x] Единая цветовая палитра, соответствующая теме.
- [x] Читаемые шрифты и уместные иконки.
- [x] Адаптация под целевое устройство (web/мобильное) – нет «растянутых» кнопок и элементов.

### 8.2 Навигация (1 балл)
- [x] Удобная и интуитивная навигация между разделами.
- [x] Наличие постоянного меню (нижняя панель, боковое меню или вкладки).
- [x] Возможность вернуться назад (системная кнопка «Назад» или кнопка в приложении).

### 8.3 Формы ввода данных (1 балл)
- [x] Все формы имеют понятные метки и подсказки.
- [x] Реализована валидация (например, проверка email, обязательные поля).
- [x] При ошибках поля подсвечиваются и/или показывается сообщение.

### 8.4 Отображение данных (1 балл)
- [x] Данные представлены в удобном виде (списки, карточки, графики).
- [ ] Реализована сортировка и/или фильтрация (если уместно).
- [ ] Есть ненавязчивые уведомления об успешных действиях (Snackbar, Toast), которые исчезают автоматически.

## ⚠️ Важно! Штраф за вылеты и необработанные ошибки
- [ ] **Приложение не вылетает (0 баллов, иначе -6):** все возможные исключения обработаны (try/catch, проверка соединения, обработка ошибок Firebase). Пользователь получает понятное сообщение об ошибке, а приложение продолжает работу.
